###### Monolito - Escala de forma vertical, não horizontal
- Tem diversos módulos dentros
- Se tiver uma indisponibilidade em um ponto dentro dele, é gerado disponibilidade em tudo
- ERP -> Geralmente é um monolitão

###### Monolito Modular - Modulos são criados deforma separadas
 - Comunicação de modulos geralmente em protocolos ou arquivos
 - As vezes um tem dependencia do outro
 - Compartilham a mesma base de dados local ou não

###### Microsserviços - Eles são independentes um do outro

- Cada um tem sua base, banco de dados
-  trabalham independente do outro
- Independente não compartilham nada
Pode acontecer um [[Anotações Arq curso Udemy#^9c0727|Event bus]] ou um processo de eventos (um barramento) fazer essa comunicaçãoe todos ms fazerem [[Anotações Arq curso Udemy#^a2fc15|Pull Push]]
- Esses MS tem dados
- Cada ms tem o seu contexto

- Pode ser diferentes camadas

- que podem ser EKS dentro da aws

- Cada aplicação rodando dentro desses "EC2" (servidores)
- Se um server cai, voce precisa lidar com um servidor novo
Por isso que aí voce pode começar a trabalhar com algo chamada

EKS FARGATE
