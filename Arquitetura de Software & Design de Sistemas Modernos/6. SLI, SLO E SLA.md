### **SLI – o que você mede**

É o **número real**, a métrica.

Exemplos:

- Tempo de resposta da API
    
- Porcentagem de uptime
    
- Taxa de erro
    

 _“Quanto o sistema está performando de verdade?”_

---

### **SLO – o objetivo**

É a **meta interna** baseada no SLI.

Exemplos:

- “A API deve ficar disponível **99,9% do tempo**”
    
- “95% das requisições devem responder em até **300 ms**”
    

 _“O quanto a gente quer que o sistema seja bom”_

---

### **SLA – a promessa**

É o **acordo com o cliente**.  
Se não cumprir, **tem consequência** (multa, crédito, penalidade).

Exemplo:

- “Garantimos **99,5% de disponibilidade mensal**”
    

 _“O que prometemos oficialmente para o cliente”_

---

### Resumo em uma frase

- **SLI** → mede
    
- **SLO** → objetivo interno
    
- **SLA** → promessa formal ao cliente
    

---

### Exemplo rápido

- **SLI**: uptime medido = 99,92%
    
- **SLO**: meta interna = 99,9%
    
- **SLA**: prometido ao cliente = 99,5%
    

Mesmo se o sistema falhar um pouco, você ainda pode:

- bater o **SLO**
    
- cumprir o **SLA**
    
- e evitar penalidade
