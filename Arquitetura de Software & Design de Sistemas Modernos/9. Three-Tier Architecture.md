Trhee Tier - Três camadas
- Ou seja temos essa aplicação dividida em três partes 

- A Primeira camada -> Chama uma API, via rede
- A segunda camada - Armazena o dado
- A Terceira camada tem o dado armazenado 
- A terceira camada passa a resposta para o back que volta para o front e retorna para o requisitante 

###### Exemplo:
- Frontend 
	- Site de vendas
	- Clica no botão Fechar pedidos
		- Ele vai chamar o backend
		- https://.... (Api que ela esta exposta)
		- vai falar para o back que fez a venda
- Backend 
	- Vai processar a venda
	- Fazer um put na base de dado
- Datastore
	- Vai retornar 200 (sucess/ok) para o back
- Backend
	- Vai retornar 200 para o front
- Frontend 
	- Vai retornar 200 para o requisitante
- Requisitante
	- Pedifo fechado 
	- "Compra com sucesso"

chamada do front para o back e o back para a base dados geralmente acontece por Rest. MEtodo Http
